AnnuAI — Présentation du projet

.NET MAUI Desktop (Windows) • MVVM • EF Core SQLite

1. Contexte & Objectif

  • Créer un annuaire d’entreprise moderne.
  • Mode visiteur + mode administrateur.
  • Données locales SQLite pour démo rapide.

2. Périmètre fonctionnel

  • Recherche salariés.
  • Gestion sites/services/salariés.
  • Import RandomUser.
  • Export fiche salarié en PDF.

3. Stack technique

  • .NET MAUI (UI desktop).
  • CommunityToolkit.Mvvm (MVVM).
  • EF Core + SQLite (persistance).
  • PdfSharpCore (PDF).

4. Architecture

  • Folders: Models, ViewModels, Views, Services, Data.
  • Injection de dépendances dans MauiProgram.
  • Séparation claire UI / logique / données.

5. Modèle de données

  • Entités: Salarie, Site, Service.
  • Relations: salarié → site, salarié → service.
  • Seed de données françaises.

6. Parcours visiteur

  • Recherche par nom/prénom/email.
  • Filtres par site et service.
  • Affichage détail salarié.

7. Export PDF

  • Bouton “Exporter en PDF”.
  • Génération fiche salarié complète.
  • Sauvegarde dans dossier AppData/exports.

8. Accès administrateur

  • Raccourci clavier Ctrl + Shift + A.
  • Page de connexion admin.
  • Logs succès/échec d’authentification.

9. Dashboard Admin

  • 3 onglets: Sites, Services, Salariés.
  • CRUD complet pour chaque entité.
  • Validations champs obligatoires + refresh listes.

10. Import RandomUser

  • Appel API randomuser.me (nat=fr).
  • Transformation en salariés.
  • Affectation auto site/service + insertion SQLite.

11. Qualité & Robustesse

  • Gestion d’erreurs centralisée par service/viewmodel.
  • Fichier de logs texte.
  • SQLite configurée pour réduire conflits multi-instance.

12. Démonstration (15 min)

  • Visiteur → Export PDF → Admin → CRUD → Import API.
  • Montrer logs + configuration technique.
  • Conclure sur apports et limites.

13. Conclusion & perspectives

  • Projet fonctionnel et démontrable.
  • Prochaines étapes: tests auto, sécurité renforcée, CI/CD.
  • Merci.
